SMACSS REFACTOR - TECHNICAL DOCUMENTATION
Project: HH.ru (Vite + React, global SCSS)
Date: 2026-02-16

1) OBJECTIVE
- Reorganize global SCSS into SMACSS layers without visual regressions.
- Improve maintainability, reduce cascade complexity and style coupling.
- Standardize naming for layout/state/module layers.

2) INITIAL CONSTRAINTS
- Global styles imported once in application entrypoint.
- SCSS compilation stays centralized through main.scss.
- Existing UI behavior preserved where possible.

3) ENTRYPOINT AND LOAD ORDER
- Global import kept in: src/app/entry/main.jsx
  - import "@/shared/styles/main.scss"
- main.scss updated to stable layer order:
  1. Base
  2. Theme
  3. Layout
  4. Modules
  5. State

4) SMACSS LAYER IMPLEMENTATION

4.1 Base
- base/reset.scss: browser reset rules.
- base/tags.scss: tag-level defaults (html/body/footer).
- base/global.scss: global utilities only (box-sizing, focus-ring helper).
- Component-specific styling removed from Base.

4.2 Theme
- Introduced token layer:
  - src/shared/styles/theme/tokens.scss
- Added CSS custom properties:
  - :root, [data-theme="dark"]
  - [data-theme="light"]
- Existing SCSS variables now mapped to CSS variables for compatibility:
  - $primary-color, $color-accent, $background-color, etc.
- Theme forwarding normalized through:
  - theme/index.scss
  - theme/colors.scss -> forward tokens

4.3 Layout
- New layout layer:
  - src/shared/styles/layout/index.scss
  - src/shared/styles/layout/app.scss
  - src/shared/styles/layout/pages.scss
- Added layout classes:
  - .l-page
  - .l-page__content
  - .l-page-section
  - .l-scroll-panel
  - .l-center
- Page wrappers were extended with layout classes in JSX model constants.

4.4 Modules
- New modules aggregator:
  - src/shared/styles/modules/index.scss
- Module files remain in components/ and are consumed via modules/index.scss.
- Components migrated toward explicit block naming and lower specificity.

4.5 State
- New state layer:
  - src/shared/styles/state/index.scss
  - src/shared/styles/state/visibility.scss
  - src/shared/styles/state/interactive.scss
- Added state classes:
  - .is-open
  - .is-hidden
  - .is-active
  - .is-disabled
  - .has-error
- Runtime-controlled states wired in JSX (modals, dot navigation, video states).

5) KEY CLASSNAME MIGRATIONS

5.1 Header
- menu__list -> header__nav-list
- menu__item -> header__nav-item
- menu__item--link -> header__nav-link
- nav wrapper -> header__nav

5.2 Footer
- menu__list -> footer__nav-list
- menu__list--link -> footer__nav-link
- logo -> footer__logo
- copyright__text -> footer__copyright
- copyright-link -> footer__copyright-link

5.3 Info block
- info-container -> info__marker
- info-icon -> info__marker-icon

5.4 Contact modal
- contactMeModal* -> contact-modal*

5.5 State-driven controls
- dotnav__dot--active -> is-active
- video state modifiers -> is-loading / is-empty / is-loading-more / has-error

6) JSX/LOGIC UPDATES
- Layout root now includes:
  - className="l-page container"
  - data-theme="dark"
- ModalWindow no longer conditionally unmounts immediately:
  - state classes control visibility (.is-open / .is-hidden)
- Images and Video modals receive .is-open class when active.
- DotNav active state switched to generic state class.

7) PAGE/LAYOUT NORMALIZATION
- Home, Projects, Contacts, Skills, About, NotFound sections aligned with l-* layout classes.
- Scrollable page panels normalized through .l-scroll-panel where applicable.
- Repeated hide/show style fragments removed from page-level SCSS.

8) SPECIFICITY/CASCADE IMPROVEMENTS
- Reduced deep nesting in page styles.
- Moved visibility/interactivity overrides to dedicated state layer.
- Module styles rely on tokenized values rather than hard-coded colors.
